# ИНСТРУКЦИЯ ДЛЯ СЕРВИСА TRANSFORMER (CPU-bound)

## 1. Настройки Воркера
- Конфигурация `arq`: `max_jobs = 1` (защита CPU от перегрузки).
- В `on_startup` один раз инициализировать `SentenceTransformer('cointegrated/rubert-tiny2')` и закодировать резюме из `settings.TARGET_RESUME` в эталонный вектор (хранить в памяти воркера).

## 2. Конвертация и Дедупликация контента
- Сырой HTML переводить в Markdown через `MarkItDown`.
- Вычислять SHA-256 от Markdown. Если хэш уже есть в БД — привязать текущую ссылку к старой вакансии и завершить таску. Хэшировать HTML запрещено.

## 3. Умный Чанкинг и Фильтрация MaxSim
- Бить Markdown на строки (`\n`). Собирать в чанки макс. длиной 1200 символов. Нахлест при закрытии чанка: `overlap_lines = 2`.
- Считать косинусное сходство каждого чанка с вектором резюме в памяти через `numpy`.
- Если `max_similarity < 0.70` -> статус ссылки `rejected` -> стоп.
- Если `>= 0.70` -> сохранить вектор лучшего чанка (размерность 312) в `vacancies.embedding` -> отправить ID в `ai_queue`.
